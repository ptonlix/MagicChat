# 部署形态

日期：2026-07-01

## 核心结论

产品默认不是 SaaS，而是独立部署形态。

客户需要先安装一个服务端，在服务端完成组织、用户、权限、模型、Agent、外部账号和企业系统集成配置。员工再通过客户端登录使用。

这个选择符合产品的信任前提：用户会授权邮件、IM、联系人、任务、HR、财务、客户等敏感数据给 AI 助手处理。如果是纯 SaaS，很多企业会天然抗拒。

## 基本形态

产品由几部分组成：

1. 服务端：企业安装和运维的核心系统。
2. 管理端：管理员配置组织、用户、权限、连接器、模型、Agent 和审计策略。
3. 客户端：员工日常使用的入口，可以是桌面端、移动端和 Web 客户端。
4. 连接器：连接邮件、钉钉、飞书、企业微信、Slack、Teams、日历、任务和文档系统。
5. 模型网关：统一接入云模型、私有模型或本地模型。
6. Agent 运行层：负责个人助理、团队 Agent、工具调用、后台任务和审计记录。

## 服务端职责

服务端应负责：

- 组织和用户管理。
- 登录、会话和设备管理。
- 角色、权限和策略管理。
- 消息、话题、任务和 Agent 状态存储。
- 外部账号授权和连接器管理。
- 模型供应商配置。
- Agent 创建、安装、启停和权限配置。
- 工具调用和后台任务执行。
- 敏感数据识别、脱敏、审批和审计。
- 日志、备份、升级和健康检查。

## 客户端职责

客户端应负责：

- 用户登录。
- 与个人 AI 助理对话。
- 与人、群组和 Agent 对话。
- 查看通讯录、联系人、Agent 和外部 IM 联系人。
- 查看和管理 todo / 任务。
- 接收通知。
- 展示待确认动作，例如待发送邮件、待回复消息、待审批 Agent 操作。

客户端不应直接保存过多敏感数据。核心数据和策略应由服务端控制。

## 管理端职责

管理员需要能配置：

- 组织结构。
- 用户和角色。
- 可接入的外部渠道。
- 每个渠道的读写权限。
- 哪些模型可用。
- 是否允许使用外部云模型。
- Agent 可调用哪些工具。
- 哪些动作必须人工确认。
- 敏感数据策略。
- 审计日志和数据保留策略。

## 模型部署选择

独立部署不等于必须本地模型。可以支持三种模式：

1. 云模型模式：服务端通过企业 API key 调用 OpenAI、Anthropic、Google、DeepSeek、阿里云、火山等模型。
2. 私有模型模式：服务端调用企业内网模型服务。
3. 本地模型模式：服务端或内网 GPU 运行本地模型。

管理员应能按数据敏感度配置模型路由：

- 普通消息可以走云模型。
- HR、财务、法务、客户机密只能走私有或本地模型。
- 某些高敏数据只允许规则处理，不进入模型。

## 连接器部署

连接器可以有两种形态：

1. 服务端连接器：服务端统一连接邮件、IM、日历和企业系统。
2. 用户级连接器：用户自己授权个人邮箱、个人 IM 或个人日历。

企业管理员需要能控制：

- 是否允许用户自行添加连接器。
- 哪些连接器可用。
- 连接器能读取哪些数据。
- 连接器能不能发送消息。
- 出站动作是否需要审批。

## 数据边界

独立部署的核心价值是数据边界清晰：

- 企业数据默认留在企业服务端。
- 外部模型调用必须经过模型网关。
- 每次敏感数据访问都可审计。
- Agent 不能绕过服务端直接访问外部系统。
- 客户端只通过服务端授权 API 操作。

## 优势

独立部署带来的优势：

- 更容易获得企业信任。
- 适合邮件、IM、HR、财务、客户数据等敏感场景。
- 企业可以选择模型供应商或私有模型。
- 权限、审计、数据保留、备份和网络策略由企业控制。
- 更适合做深度企业系统集成。

## 成本与风险

独立部署也会带来成本：

- 安装和升级复杂度更高。
- 需要支持多种企业网络环境。
- 连接器调试和权限配置更复杂。
- 客户需要运维能力。
- 需要提供健康检查、备份、迁移、监控和故障排查工具。
- 销售周期可能比自助 SaaS 更长。

## 产品含义

这个部署形态会影响产品定位：

- 更适合企业和高安全要求团队。
- 不适合一开始做纯个人消费产品。
- 可以强调“企业自有 AI 工作入口”。
- 可以把安全、权限、审计、模型可控作为核心卖点。
- MVP 需要有足够简单的安装和配置体验，否则试用门槛会太高。

## MVP 建议

第一版可以采用轻量独立部署：

- Docker Compose 安装。
- 单服务端 + 数据库 + Redis / 队列。
- 系统内置管理员账号。
- 管理员创建用户，不开放用户注册。
- 管理员配置一个模型供应商。
- 用户通过 Web 客户端登录。
- 先支持通讯录、内置 AI 助手对话、用户一对一聊天。

后续再扩展：

- Kubernetes 部署。
- 高可用。
- 多租户或多组织。
- 私有模型路由。
- 更多 IM 连接器。
- 桌面端和移动端。
