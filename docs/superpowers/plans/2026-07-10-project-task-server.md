# Project and Task Server Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Deliver authenticated project, project-group, dynamic-member, and task APIs, including an undeletable personal workspace for every user.

**Architecture:** Add explicit PostgreSQL tables and matching GORM models, then expose handlers under the existing `/api/client` session-protected group. Project access is derived at query time from ownership or active human membership in an active linked group; task mutations use that same access predicate and update the parent project's activity timestamp transactionally.

**Tech Stack:** Go 1.25, Echo, GORM, PostgreSQL, Goose, SQLite handler tests, Swaggo.

```go
func normalizeTaskLabels(labels []string) ([]string, error) {
	if len(labels) > 20 { return nil, errInvalidTaskLabels }
	seen := map[string]struct{}{}
	result := make([]string, 0, len(labels))
	for _, raw := range labels {
		label := strings.TrimSpace(raw)
		if utf8.RuneCountInString(label) < 1 || utf8.RuneCountInString(label) > 32 {
			return nil, errInvalidTaskLabels
		}
		key := strings.ToLower(label)
		if _, exists := seen[key]; exists { continue }
		seen[key] = struct{}{}
		result = append(result, label)
	}
	return result, nil
}
```
