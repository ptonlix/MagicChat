# MyGod

MyGod 是一个面向企业团队的 AI 原生工作入口。它希望让每个人都拥有自己的工作助理，让 AI 不只是被动回答问题，而是能理解消息、整理上下文、提取任务、推进工作，并在需要人工判断时把事情交还给人。

## 为什么做

今天的企业协作仍然围绕聊天、群组、邮件、会议和任务列表展开。人需要不断读消息、分流请求、追踪上下文、确认进展、催办协作。

MyGod 想解决的是这个问题：让 AI 助理先接住工作请求，把杂乱的沟通整理成可以处理的事项，让人从重复的信息处理里解放出来。

## 产品定位

MyGod 不是简单的聊天工具，也不是给 IM 加一个机器人。

它更像一个“助理优先”的企业工作入口：

- 每个用户都有自己的个人 AI 助理。
- 团队成员可以和人沟通，也可以和 Agent 协作。
- 消息、任务、上下文和执行记录可以沉淀在同一个工作空间里。
- AI 可以先处理、总结、草拟、分流和跟进，再把重要决策交给人确认。

## 当前阶段

MyGod 目前还处在产品原型和能力验证阶段，重点是先跑通企业协作的基础体验：

- 用户登录自己的工作空间。
- 查看团队通讯录。
- 与内置 AI 助手对话。
- 与团队成员聊天。
- 逐步完善个人资料、头像和工作身份。
- 管理员可以管理用户、基础设置和登录方式。

## 未来方向

长期来看，MyGod 希望成为企业里的 AI 工作控制层。

邮件、消息、会议、任务和业务系统里的请求，可以先由个人助理理解、总结、分流和草拟处理；团队中的 Agent 也可以像成员一样参与讨论、承担任务、留下执行记录，并遵守清晰的权限和隐私边界。

## 核心理念

- 助理优先：工作请求先被助理理解和整理，而不是直接打断人。
- 人机协作：AI 是团队里的协作者，不只是工具按钮。
- 上下文沉淀：重要沟通、决策和任务不应该散落在消息流里。
- 权限清晰：AI 的行动必须有边界，敏感信息和高风险动作需要可控。
- 企业可控：企业应该能掌握自己的数据、模型、权限和集成方式。
